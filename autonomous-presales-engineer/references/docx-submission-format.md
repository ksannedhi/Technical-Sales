# DOCX Submission Formatting Standard

## Goal
Produce DOCX outputs that are ready for customer-facing bid submission, not raw markdown conversions.

## Required Elements
1. Professional title page or cover block.
2. Document control block: prepared for/by and date.
3. Confidentiality label.
4. Auto-generated Table of Contents (Heading 1-3).
5. Page numbering in footer.
6. Consistent heading/body styles.
7. Structured tables for BOQ, SLA, and risks.

## Figure Policy
1. Insert figures that support architecture, service model, or workflow understanding.
2. Prefer vendor-provided visuals from source proposal files.
3. Add short captions for each figure.
4. Avoid decorative or redundant visuals.

## Cleanliness Checks
1. No markdown symbols left in visible document text.
2. No broken image placeholders.
3. TOC updates correctly and headings map properly.
4. Footer page numbers render across pages.
